# حل مشكلة الاتصال بـ Supabase

## المشكلة
الموقع لا يعمل بعد تغيير الموديم أو مزود الإنترنت بسبب مشكلة DNS.

## الحل السريع

### الطريقة 1: تشغيل ملف الإصلاح التلقائي
1. انقر بزر الماوس الأيمن على ملف `fix-dns.bat`
2. اختر "تشغيل كمسؤول" (Run as Administrator)
3. انتظر حتى يكتمل الإصلاح
4. شغل الموقع: `php artisan serve`

### الطريقة 2: تغيير DNS يدوياً
1. افتح **لوحة التحكم** → **الشبكة والإنترنت** → **مركز الشبكة والمشاركة**
2. اضغط على اتصال الشبكة النشط
3. اضغط **خصائص** (Properties)
4. اختر **Internet Protocol Version 4 (TCP/IPv4)**
5. اضغط **خصائص**
6. اختر **استخدام عناوين خادم DNS التالية**:
   - **المفضل**: `8.8.8.8`
   - **البديل**: `8.8.4.4`
7. اضغط **موافق**

### الطريقة 3: استخدام أمر واحد
افتح Command Prompt كمسؤول ونفذ:
```cmd
ipconfig /flushdns && netsh interface ipv4 set dns "Wi-Fi" static 8.8.8.8
```

## بعد حل المشكلة
```bash
php artisan config:clear
php artisan serve
```

## ملاحظة مهمة
- الموقع الآن يستخدم **Supabase Connection Pooler** (Port 6543)
- يعمل مع جميع أنواع الإنترنت بعد تغيير DNS
- البيانات محفوظة في Supabase ولن تضيع

## إذا استمرت المشكلة
تواصل مع مزود الإنترنت وتأكد أنه لا يمنع الاتصالات الخارجية على Port 6543
